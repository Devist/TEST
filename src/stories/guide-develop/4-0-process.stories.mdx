### Step1. 이슈 등록 및 검토 요청

본인이 맡은 개발 영역에 대해 1~2일 단위의 모듈로 나누어 Jira 이슈 등록하며,

**Due Date / Labels / Fix Version/s Assignee 반드시 등록합니다**.

- Jira 이슈 등록시 Description에 아래 추가

```
[checkbox]테스트케이스 작성

[checkbox]테스트케이스 검토

[checkbox]구현

[checkbox]테스트 진행 (본인)

[checkbox]테스트 진행 (검토자)

[checkbox]코드 리뷰 통과

구현 방안 >
```

### Step2. 엔티티 및 서비스 레이어 설계 > 테스트 > 구현

테스트케이스를 스프레드시트에 작성 완료한 후 검토까지 완료되었다면, Entity와 Service를 설계 > 테스트 > 구현합니다.

이 때, 단위테스트를 통해 비즈니스 로직을 검증합니다.

```javascript
// 단위 테스트 실행 스크립트
yarn test:unit
```

### Step3. 스토어 및 UI 레이어 구현 > 테스트

스토어 및 UI 레이어를 구현한 후 ,

- 구글 스프레드시트에 사전 작성한 테스트를 진행합니다.
- 이 때, 일부 테스트는 E2E 테스트 자동화 코드를 작성하여 진행합니다.
- 이 후 검토자에게 테스트 요청합니다.
  - 검토자의 테스트는 개발단계에서는 생략될 수 있습니다. - 기한에 맞추기 위해

```javascript
// E2E 테스트 실행 스크립트
yarn test:e2e:local
```

### Step4. 코드리뷰

코드리뷰를 진행하며 나온 개선사항에 대하여 반영 후, 테스트를 1회 더 진행합니다.

- 운영 단계 이후의 추가 기능 개발, 버그 픽스 등에선 필수
- 개발 단계에서는 생략 가능 - 기한에 맞추기 위해

**특수한 상황이 아닌 경우, 작업 단위로 push 합니다.**
